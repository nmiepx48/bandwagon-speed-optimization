## 搬瓦工速度慢，90% 的原因在机房线路

先给一个定义：**搬瓦工速度**，本质上是你到 VPS 服务器之间的网络链路质量。同样的机器配置，不同线路，体验可以差出好几倍。

买搬瓦工的国内用户，访问速度受两个因素决定：

1. **地理距离**：服务器在哪，物理上离你多远
2. **线路等级**：走的是什么网络，高峰期有没有拥堵保障

这两个里，线路等级的影响更大。一台香港服务器走烂线路，可能还不如洛杉矶走 CN2 GIA 稳。

## 搬瓦工有哪些线路？速度差在哪

搬瓦工目前主推的线路分三档，从低到高：

**普通国际线路（KVM 套餐默认机房）**

弗里蒙特、洛杉矶 DC2/DC4/DC8 这些机房走的是常规国际线路。带宽便宜，但晚高峰丢包率高，国内访问体感明显不稳定。适合学习 Linux 或跑不需要对国内访问的程序，拿来建站给国内用户访问就比较吃力。

**CN2 GT 线路（入门优化）**

走中国电信的 CN2 GT 通道，相当于在出境那一段走了快速路，但进入国内各省还是普通路。日常用体验尚可，晚高峰 8-11 点延迟会往上涨，丢包偶尔出现。

**CN2 GIA 线路（最高优先级）**

CN2 GIA（Global Internet Access）是中国电信最高等级的民用优化线路，全程高优先级通道，不和普通流量抢带宽。三网（电信、联通、移动）回程都走 CN2 GIA，晚高峰基本不会出现明显拥堵。这就是为什么用过的人通常会提到"晚上也不卡"——线路本身就在高峰期保住了稳定性。

搬瓦工的 CN2 GIA 线路目前主要分两类：
- **CN2 GIA-E（电商版）**：洛杉矶 DC6/DC9 机房，美加多个 CN2 GIA 机房可切换，三网回程 CN2 GIA，带宽从 2.5Gbps 起步
- **香港/东京 CN2 GIA**：亚太机房，物理距离近，延迟更低，价格也更高

用国内三大运营商的情况不一样，机房选择也有区别：

**电信用户**

DC6 CN2 GIA-E 或 DC9 CN2 GIA 都是最优选择。电信走 CN2 GIA 回程，延迟大概在 130-160ms 区间（洛杉矶到国内），晚高峰也保持稳定。

## 除了换机房，还有几件事可以做

机房是最核心的，但还有一些配置层面的东西能帮你再榨出一些速度：

**开启 BBR 加速**

BBR 是 Google 开发的 TCP 拥塞控制算法，在高延迟、高丢包环境下效果明显。搬瓦工 VPS 支持自行安装，Linux 内核 4.9 以上版本都可以直接启用。在 KiwiVM 里可以一键切换新版系统，然后用几行命令开启 BBR。具体命令因系统版本而异，网上有现成脚本，照着跑一遍就行。

**选一个距离你最近的机房反复测试**

测延迟最简单：ping 一下各机房的测速 IP。搬瓦工各机房都有官方测速节点，KiwiVM 面板里可以找到。不要只看一次结果，白天和晚高峰各测一次，差距可能很明显。

**查一下当前 IP 是不是被污染**

有时候速度差不是线路问题，是 IP 本身被某些规则拦截了。可以在 KiwiVM 里申请更换 IP（有次数限制），换完再测对比。

**看流量是不是快用完了**

搬瓦工各套餐都有月流量上限，超出后会降速或暂停服务。KiwiVM 面板里可以看当月已用流量，如果快到上限了，这也可能是速度变慢的原因之一。
